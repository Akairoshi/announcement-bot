# Бот для объявлений (AnnouncementBot)

## Описание

Бот рассылает уведомления по категориям. Администраторы создают объявления для своих категорий, пользователи подписываются и получают уведомления.

---

## Роли

### Пользователь

Может подписываться на категории и получать уведомления.

| Команда | Описание |
|---------|----------|
| `/start` | Активация бота |
| `/profile` | Просмотр профиля |
| `/subscribe` | Подписка / отписка от категорий |
| `/list_announcement` | Просмотр последних уведомлений по своим категориям |
| `/admin_request` | Заявка на роль администратора |

### Администратор

Назначается SuperAdmin'ом. Создаёт объявления для своих категорий, управляет личными шаблонами.

| Команда | Описание |
|---------|----------|
| `/profile` | Просмотр профиля |
| `/make_announcement` | Создать объявление для своих категорий |
| `/list_announcement` | Просмотр объявлений по своим категориям |
| `/list_template` | Список своих шаблонов |
| `/add_template` | Создать шаблон |
| `/update_template` | Изменить шаблон |
| `/remove_template` | Удалить шаблон |

### Супер Администратор

Primary СА назначается через конфиг — не может быть разжалован через бота. Управляет категориями, администраторами и заявками.

| Команда | Описание |
|---------|----------|
| `/profile` | Просмотр профиля |
| `/make_announcement` | Создать объявление для своих категорий |
| `/list_announcement` | Просмотр всех объявлений |
| `/list_admin_request` | Просмотр заявок на администрирование |
| `/list_admin` | Список администраторов |
| `/remove_admin` | Удалить администратора |
| `/list_category` | Список категорий |
| `/add_category` | Добавить категорию |
| `/update_category` | Изменить категорию |
| `/remove_category` | Удалить категорию |
| `/list_template` | Список своих шаблонов |
| `/add_template` | Создать шаблон |
| `/update_template` | Изменить шаблон |
| `/remove_template` | Удалить шаблон |

## Сущности БД

| Сущность | Поля |
|----------|------|
| `Users` | Id, UserName, Role, JoinAt |
| `Categories` | Id, Name, CreatedById, CreatedAt |
| `AdminCategoryAccess` | AdminId, CategoryId |
| `Subscriptions` | UserId, CategoryId, SubscribedAt |
| `Templates` | Id, Name, Text, CreatedById, CreatedAt |
| `Announcements` | Id, Text, CategoryId, TemplateId, CreatedById, CreatedAt |
| `DeliveryStatus` | Id, AnnouncementId, UserId, Status, RetryCount, LastAttemptAt, SentAt |
| `AdminRequests` | Id, RequesterId, TargetId, Type, Details, Status, CreatedAt, ReviewedAt, ReviewedById |
| `AuditLog` | Id, UserId, Action, EntityName, EntityId, Details, CreatedAt |

**Автоматика:**
- `Announcements` — удаляются через 30 дней (`CleanerService`, раз в сутки)
- `DeliveryStatus` — повторная отправка каждые 10 минут, до 3 попыток (`SenderService`)
- `AuditLog` — заполняется автоматически через перехват `SaveChanges` в EF Core

## Flow команд

### `/start`
- Показывает информацию о боте и предлагает подписаться через `/subscribe`

---

### `/profile`
- Получает данные пользователя из БД
- Получает список подписанных категорий из БД
- Выводит информацию о пользователе и его категориях

---

### `/subscribe`
- Выводит список всех категорий кнопками (получение из БД)
- Пользователь выбирает категории:
  - Если подписка есть → отменить (удалить из БД)
  - Если подписки нет → добавить (записать в БД)

---

### `/list_announcement`
- **Пользователь** — выводит объявления из своих подписанных категорий
- **Администратор** — выводит объявления из своих назначенных категорий
- **SuperAdmin** — выводит все объявления

---

### `/make_announcement`
1. Выбор категории кнопками (категории доступные администратору, из БД)
   - Категорий нет → ошибка
2. Выбор шаблона кнопками (шаблоны администратора, из БД) или продолжение без шаблона
3. Ввод данных:
   - Без шаблона — вводится текст вручную
   - С шаблоном — заполняются плейсхолдеры поочерёдно
4. Предпросмотр и подтверждение
5. Сохранение объявления в БД
6. Создание записей рассылки в `DeliveryStatus`

---

### `/add_template`
1. Ввод названия шаблона
2. Ввод текста шаблона
3. Сохранение в БД

---

### `/update_template`
1. Выбор шаблона кнопками (получение из БД)
   - Шаблонов нет → ошибка
2. Ввод нового названия и/или текста
3. Обновление записи в БД

---

### `/remove_template`
1. Выбор шаблона кнопками (получение из БД)
   - Шаблонов нет → ошибка
2. Подтверждение удаления
   - Да → удалить из БД
   - Нет → отмена

---

### `/list_template`
- Выводит список шаблонов текущего администратора (получение из БД)

---

### `/add_category`
1. Ввод названия категории → проверка уникальности в БД
   - Уже существует → ошибка
   - Не существует → сохранить в БД

---

### `/update_category`
1. Выбор категории кнопками (получение из БД)
   - Категорий нет → ошибка
2. Ввод нового названия → проверка уникальности в БД
   - Уже существует → ошибка
   - Не существует → обновить запись в БД

---

### `/remove_category`
1. Выбор категории кнопками (получение из БД)
   - Категорий нет → ошибка
2. Подтверждение удаления
   - Да → удалить из БД
   - Нет → отмена

---

### `/list_category`
- Выводит список всех категорий (получение из БД)

---

### `/list_admin`
- Выводит список всех администраторов (получение из БД)

---

### `/remove_admin`
1. Выбор администратора кнопками (получение из БД)
   - Администраторов нет → ошибка
2. Подтверждение удаления
   - Да → понизить до User в БД
   - Нет → отмена

---

### `/admin_request`
1. Выбор типа заявки: **Назначение** / **Переназначение**

**Назначение** (пользователь хочет стать Admin):
1. Ввод причины (Details)
2. Сохранение заявки в БД

**Переназначение** (Admin хочет передать роль другому):
1. Ввод username или ID пользователя → проверка существования в БД
   - Не найден → ошибка
2. Ввод причины (Details)
3. Сохранение заявки в БД

---

### `/list_admin_request`
- Выводит список заявок кнопками (получение из БД)
- SuperAdmin выбирает заявку и принимает решение:

**Назначение:**
- Одобрить:
  1. Выбор категории кнопками (получение из БД) или создание новой
  2. Записать `AdminCategoryAccess` в БД
  3. Статус заявки → `Approved`
- Отклонить:
  1. Статус заявки → `Rejected`
  2. Пользователь получает уведомление об отказе

**Переназначение:**
- Одобрить:
  1. Новый пользователь получает роль Admin с категориями подавшего
  2. Подавший заявку понижается до User
  3. Статус заявки → `Approved`
- Отклонить:
  1. Статус заявки → `Rejected`
  2. Пользователь получает уведомление об отказе
