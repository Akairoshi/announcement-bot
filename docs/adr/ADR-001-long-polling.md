# ADR-001: Long Polling вместо Webhook

- **Статус:** Принято
- **Дата:** 2025-05-17

---

## Контекст

Telegram Bot API поддерживает два способа получения обновлений:

- **Webhook** — Telegram сам отправляет POST-запрос на указанный URL при каждом обновлении
- **Long Polling** — бот сам периодически запрашивает обновления у Telegram API

Бот разворачивается на выделенном ПК без публичного домена и SSL-сертификата. Webhook требует публично доступный HTTPS-эндпоинт — настройка домена и сертификата выходит за рамки задачи.

---

## Решение

Использовать **Long Polling** через `Telegram.Bot` с `IHostedService`.

Хост строится через `Host.CreateDefaultBuilder` без `Microsoft.NET.Sdk.Web` — HTTP-сервер не поднимается, публичный URL не нужен.

---

## Альтернативы

### Webhook + ngrok

Локальный туннель через ngrok даёт публичный URL без сервера. Подходит для разработки, но нестабилен для продакшна: URL меняется при перезапуске (в бесплатном плане), зависимость от стороннего сервиса.

### Webhook на выделенном ПК

Потребовало бы: настройку домена, получение SSL-сертификата, открытие порта, подъём HTTP-сервера. Избыточно для текущего масштаба проекта.

---

## Последствия

- Проект не требует `Microsoft.NET.Sdk.Web` и ASP.NET Core
- Обработка Update'ов реализуется через `IHostedService` + `ReceiverOptions`
- При масштабировании или переносе на сервер с доменом — миграция на Webhook потребует замены одного сервиса и смены SDK, остальная архитектура не затрагивается
