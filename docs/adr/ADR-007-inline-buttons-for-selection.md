# ADR-007: Выбор сущностей через инлайн-кнопки вместо текстового ввода

- **Статус:** Принято
- **Дата:** 2026-06-01

---

## Контекст

Изначально ряд команд предполагал выбор сущностей через текстовый ввод — пользователь вводил название категории, username администратора, ID шаблона. Это создавало проблемы:

- Легко ошибиться в названии
- Нет подсказок — пользователь не знает что именно вводить
- Плохой UX — особенно на мобильных устройствах

---

## Решение

Везде где нужно выбрать существующую сущность — показывать инлайн-кнопки с актуальными данными из БД.

| Операция | Было | Стало |
|---|---|---|
| Выбор шаблона для объявления | Текст (ID) | Инлайн-кнопки |
| Выбор категории для объявления | Текст (название) | Инлайн-кнопки |
| Выбор категории для удаления | Текст (название) | Инлайн-кнопки |
| Выбор категории для переименования | Текст (название) | Инлайн-кнопки |
| Выбор шаблона для обновления | Текст (ID) | Инлайн-кнопки |
| Выбор шаблона для удаления | Текст (ID) | Инлайн-кнопки |
| Выбор администратора для удаления | Текст (username) | Инлайн-кнопки |

Текстовый ввод остаётся только для **новых данных** — названия, текста, причины.

---

## Альтернативы

### Оставить текстовый ввод

Проще в реализации, не требует дополнительных запросов к БД при старте команды. Но плохой UX — пользователь должен знать точные названия/ID наизусть.

---

## Последствия

- FSM состояния для выбора сущностей заменяются на показ кнопок + `CallbackHandler`
- Каждая команда выбора делает запрос к БД для получения актуального списка
- Если список пустой — сообщение об отсутствии данных вместо кнопок
- Callback data должна укладываться в лимит Telegram 64 байта — используем короткие префиксы
